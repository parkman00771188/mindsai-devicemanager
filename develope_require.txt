Device Manager Local

또는

장비 대여·반납 관리 시스템

목적

회사 내부 장비, 기기, 검사 장비, VR 기기, 태블릿, 노트북, 키오스크 부품 등을 QR 코드 또는 장비번호로 조회하고, 해당 장비의 대여, 반납, 상태 변경, 사진 등록, 이력 확인을 할 수 있는 로컬 웹사이트를 만든다.

운영 방식
PC와 모바일 모두 접속 가능
로컬 서버에서 실행
접속 주소 예시: http://localhost:3000
같은 Wi-Fi 내부망에서는 PC IP로 접속 가능
예: http://192.168.0.10:3000
DB는 별도 구축하지 않고, Excel 파일을 DB처럼 사용
관리자 로그인 필요
기본 계정:
ID: admin
PW: admin123!
2. 추천 기술 스택

4. 장비 조회 방식
A. QR 코드 스캔 조회
기능
모바일 또는 노트북 카메라로 QR 코드 스캔
QR 값은 장비번호 또는 장비 상세 URL로 구성
예:
EQ-0001
또는 http://localhost:3000/device/EQ-0001
스캔 성공 시 해당 장비 상세 페이지로 이동
화면 구성
카메라 스캔 영역
“스캔 시작” 버튼
“스캔 중지” 버튼
직접 입력 검색창
최근 스캔 장비 리스트
주의사항
카메라 권한 허용 필요
로컬 테스트는 localhost 권장
내부망 모바일 접속 시 카메라 권한 이슈가 생기면 HTTPS 설정 필요 가능성 있음

이유

대여/반납 이력이 남아 있어야 하므로 실제 삭제하면 과거 기록 추적이 어려워짐.

삭제 버튼 동작

1차 MVP에서는 두 가지 옵션 제공:

폐기 처리
완전 삭제

권장 기본값:

상태를 DISPOSED로 변경

완전 삭제는 관리자 확인 팝업 후 진행.

20. Codex용 개발 지시 프롬프트

아래를 그대로 Codex에 넣으면 됩니다.

로컬에서 실행되는 장비 대여/반납 관리 웹사이트를 만들어줘.

목표:
- PC와 모바일에서 모두 사용 가능한 반응형 웹사이트
- QR 코드 스캔 또는 장비번호 검색으로 장비 조회
- 장비 대여, 반납, 상태관리, 사진 업로드, 점검/고장관리, 이력조회 기능 포함
- 별도 DB 없이 Excel 파일(.xlsx)을 DB처럼 사용
- 로컬 주소 http://localhost:3000 에서 실행
- 기본 로그인 계정은 ID admin, PW admin123!

기술 스택:
- Backend: Node.js + Express
- Frontend: React + Tailwind CSS
- Excel 처리: xlsx 또는 SheetJS
- QR 스캔: html5-qrcode
- QR 코드 생성: qrcode
- 파일 업로드: multer
- 사진 저장 위치: server/uploads/
- Excel 저장 위치: server/data/devices.xlsx

필수 페이지:
1. /login
   - ID/PW 로그인
   - 기본 계정 admin / admin123!
   - 로그인 성공 시 대시보드 이동

2. /
   - 대시보드
   - 전체 장비 수, 대여 가능, 대여 중, 점검 중, 고장 수 표시
   - 최근 대여/반납 이력 표시
   - QR 스캔, 장비 검색, 장비 등록 빠른 버튼

3. /scan
   - html5-qrcode를 이용한 QR 스캔
   - QR 값이 장비번호이면 /devices/:deviceId 로 이동
   - 직접 장비번호 입력 검색도 제공

4. /search
   - 장비번호, 장비명, 모델명, 시리얼번호, 위치로 검색
   - 검색 결과 클릭 시 상세 페이지 이동

5. /devices
   - 전체 장비 목록 테이블
   - 상태, 분류, 위치 필터
   - 장비번호, 장비명, 분류, 모델명, 위치, 상태, 현재 대여자, 반납예정일 표시

6. /devices/new
   - 장비 등록
   - 장비번호 자동 생성
   - 장비명, 분류, 제조사, 모델명, 시리얼번호, 구매일, 구매금액, 관리부서, 담당자, 위치, 구성품, 메모 입력
   - 대표 사진 업로드
   - 등록 시 QR 코드 자동 생성

7. /devices/:deviceId
   - 장비 상세 페이지
   - 장비 기본 정보, 현재 상태, 현재 대여자, 대여일, 반납예정일, 최근 점검일 표시
   - 대표 사진, QR 코드 표시
   - 대여하기, 반납하기, 수정, 점검 등록, 폐기 처리 버튼
   - 최근 대여/반납 이력 표시

8. /devices/:deviceId/edit
   - 장비 정보 수정
   - 수정 시 AuditLogs에 변경 이력 저장

9. /devices/:deviceId/rent
   - 대여 처리
   - 대여자 이름, 부서, 연락처, 사용 목적, 대여일, 반납예정일, 대여 장소, 메모 입력
   - 대여 시 사진 업로드 선택
   - 장비 상태가 AVAILABLE일 때만 대여 가능
   - 처리 후 상태를 RENTED로 변경하고 Transactions에 이력 저장

10. /devices/:deviceId/return
   - 반납 처리
   - 반납자 이름, 실제 반납일, 반납 장소, 반납 시 상태, 이상 여부, 이상 내용, 사진, 메모 입력
   - 정상 반납이면 상태를 AVAILABLE로 변경
   - 이상 있으면 MAINTENANCE 또는 BROKEN으로 변경
   - Transactions에 이력 저장

11. /transactions
   - 전체 대여/반납/상태변경 이력 조회
   - 장비번호, 사용자명, 기간, 작업유형으로 필터
   - Excel 다운로드 버튼

12. /maintenance
   - 점검/고장 이력 관리
   - 점검 등록, 고장 신고, 수리 완료 처리

13. /settings
   - Excel 다운로드
   - Excel 백업 생성
   - 초기 데이터 생성
   - 업로드 폴더 경로 표시

Excel 파일 구조:
- server/data/devices.xlsx
- Sheet는 Devices, Transactions, Maintenance, Users, AuditLogs 로 구성

Devices 컬럼:
device_id, device_name, category, manufacturer, model_name, serial_number, purchase_date, purchase_price, department, manager, location, status, current_borrower, borrower_department, borrowed_at, expected_return_at, last_returned_at, last_checked_at, components, main_photo_path, qr_code_path, memo, created_at, updated_at, is_deleted

Transactions 컬럼:
transaction_id, device_id, action_type, user_name, user_department, user_contact, purpose, rented_at, expected_return_at, returned_at, before_status, after_status, condition_status, issue_description, photo_paths, handled_by, memo, created_at

Maintenance 컬럼:
maintenance_id, device_id, maintenance_type, checked_by, checked_at, result, issue_level, action_taken, next_check_at, photo_paths, status_after, memo, created_at

Users 컬럼:
user_id, password, name, role, created_at

AuditLogs 컬럼:
log_id, user_id, action, target_type, target_id, before_value, after_value, ip_address, created_at

장비 상태:
AVAILABLE = 대여 가능
RENTED = 대여 중
MAINTENANCE = 점검 중
BROKEN = 고장
LOST = 분실
DISPOSED = 폐기

UI 요구사항:
- PC는 좌측 사이드바 + 상단 헤더 + 본문 카드형 대시보드
- 모바일은 카드형 리스트 + 큰 버튼 + 하단/상단 메뉴
- 상태별 컬러 뱃지 적용
- QR 스캔 버튼은 모바일에서 크게 표시
- 대여하기/반납하기 버튼은 장비 상세 상단에 강조
- 장비 상태가 대여 가능이 아니면 대여 버튼 비활성화
- 장비 상태가 대여 중이 아니면 반납 버튼 비활성화

보안/운영:
- MVP에서는 admin/admin123! 하드코딩 가능
- 추후 Users 시트 기반 로그인으로 확장 가능하게 작성
- 모든 변경 작업은 AuditLogs에 기록
- Excel 파일 읽기/쓰기 중 동시 저장 충돌이 나지 않도록 파일 접근 함수를 하나로 관리
- 사진 파일은 Excel에 직접 넣지 말고 경로만 저장
- 삭제는 기본적으로 완전 삭제가 아니라 is_deleted=true 또는 status=DISPOSED 처리

실행 명령어:
npm install && npm run dev
